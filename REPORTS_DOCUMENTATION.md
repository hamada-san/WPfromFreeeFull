# reports.js ドキュメント

## 概要

このスクリプトは、freee会計APIからデータを取得し、スプレッドシートの各シートに財務諸表・税務検討資料を出力するGoogle Apps Script（GAS）です。

---

## 処理フロー

```
getTrialBalanceAndPLCore() [メイン関数]
    │
    ├─→ BSシート出力
    ├─→ PLシート出力（3期比較）
    ├─→ CRシート出力（製造原価報告書・3期比較）
    ├─→ 区分別表シート出力 + PL税務検討用元帳データ作成
    ├─→ PL税務検討用元帳シート出力
    ├─→ BS税務検討用内訳シート出力
    └─→ 固定資産台帳シート出力（プラン制限あり）
```

---

## シート別出力内容

### 1. BSシート（貸借対照表）

| 項目 | 説明 |
|------|------|
| **API** | `GET /api/1/reports/trial_bs` |
| **出力位置** | B17以降 |
| **ヘッダー位置** | B16:H16 |
| **タイムスタンプ** | G15 |

#### 出力カラム（B列〜G列）

| 列 | 内容 | データソース |
|----|------|-------------|
| B | 分類 | `account_category_name` / `parent_account_category_name` |
| C | 勘定科目 | `account_item_name` |
| D | 期首残高 | `opening_balance` |
| E | 借方金額 | `debit_amount` |
| F | 貸方金額 | `credit_amount` |
| G | 期末残高 | `closing_balance` |

#### 書式設定
- G列（期末残高）: 赤色背景 `#f4cccc`
- 「資産」「負債」「純資産」行: 上部罫線

### 2. PLシート（損益計算書・3期比較）

| 項目 | 説明 |
|------|------|
| **API** | `GET /api/1/reports/trial_pl_three_years` |
| **出力位置** | B17以降 |

| **タイムスタンプ** | G15 |

#### 書式設定

### 4. 区分別表シート（税区分別集計表）

| 項目 | 説明 |
|------|------|
| **API** | `GET /api/1/deals`（取引）<br>`GET /api/1/manual_journals`（振替伝票）<br>`GET /api/1/expense_applications`（経費精算） |
| **出力位置** | B44以降 |
| **ヘッダー位置** | B43:F43 |
| **タイムスタンプ** | F42 |
| **課税方式** | E42（免税事業者/原則課税/簡易課税） |

#### 出力カラム（B列〜F列）

| 列 | 内容 | 計算方法 |
|----|------|---------|
| B | 勘定科目 | `account_item_name` |
| C | 税区分 | `tax_code`の日本語名 |
| D | 税抜金額 | 税込金額 - 税額 |
| E | 税額 | `vat`の合計 |
| F | 税込金額 | `amount`の合計 |

#### 集計ロジック
1. 取引（deals）、振替伝票（manual_journals）、経費精算（expense_applications）を取得
2. 勘定科目 × 税区分 でグループ化して金額を合計
3. BS→PLの順序（試算表の並び順）でソート

---

### 5. PL税務検討用元帳シート

| 項目 | 説明 |
|------|------|
| **データソース** | 区分別表と同時に取得（`getTaxCategoryReportCore`内） |
| **出力位置** | B17以降 |
| **ヘッダー位置** | B16:J16 |
| **タイムスタンプ** | G15 |

#### 対象勘定科目
- 雑収入
- 雑損失
- 固定資産売却益
- 固定資産売却損

#### 出力カラム（B列〜J列）

| 列 | 内容 | データソース |
|----|------|-------------|
| B | 大分類 | 勘定科目のカテゴリ |
| C | 勘定科目 | `account_item_name` |
| D | 税区分 | `tax_code`の日本語名 |
| E | 取引先タグ | `partner_id` → 名前変換 |
| F | 品目タグ | `item_id` → 名前変換 |
| G | メモタグ | `tag_ids` → 名前変換（複数は「、」区切り） |
| H | 摘要 | `description` |
| I | 借方金額 | 支出（expense）の場合の金額 |
| J | 貸方金額 | 収入（income）の場合の金額 |

#### マスタデータ取得API
- `GET /api/1/partners` → 取引先ID→名前マップ
- `GET /api/1/items` → 品目ID→名前マップ
- `GET /api/1/tags` → タグID→名前マップ

|------|------|

| 勘定科目 | 内訳タイプ |

| 未払費用 | 取引先別（partner） |
| 預り金 | 品目別（item） |
| 長期借入金 | 品目別（item） |

#### 出力カラム（B列〜H列）

| 列 | 内容 |
|----|------|
| B | 大分類 |

| E | 期首残高 |

## エラーハンドリング

| 箇所 | 処理 |
|------|------|
| CRシート | try-catch で囲み、エラー時は「製造原価報告書なし」表示 |
| BS税務検討用内訳 | try-catch で囲み、エラー時はログ出力のみ |
| 固定資産台帳 | try-catch で囲み、403/401エラー時はスキップ |
| マスタデータ取得 | 403エラー時は `_error` プロパティにエラー内容を格納 |

---

## 処理時間の最適化

1. **マスタデータ条件取得**: PL税務検討用元帳の対象科目（雑収入等）がある場合のみ、取引先・品目・タグのマスタを取得
2. **ページネーション**: 各APIは`limit=100`で分割取得
3. **並列処理なし**: GASの制約により逐次処理
